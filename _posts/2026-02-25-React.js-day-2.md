## Hook은 항상 컴포넌트 최상단에서, 매 랜더링마다 같은 순서로 호출한다.

[공식문서](https://legacy.reactjs.org/docs/hooks-rules.html)의 내용에 따르면, "반복문, 조건문 또는 중첩함수"내에서 Hooks를 호출하지 말라고 한다. 이런 이유는 무엇일까? 아래의 예시를 통해 이해한 내용을 설명하고자 한다.

```jsx
    function Form() {
        // 1. 상태저장 Hook 호출(name state)
        const [name, setName] = useState('Mary');
        // 2. 후처리 Hook 호출
        useEffect(function persistForm() {
            localStorage.setItem('formData', name);
        });

        // 3. 상태저장 Hook 호출(surname state)
        const [surname, setSurname] = useState('Poppins');

        // 4. 후처리 Hook 호출
        useEffect(function updateTitle() {
            document.title = name + '' + surname;
        });
    }
```

상기의 예시를 보면, 1~4까지 순서대로 기재되어 있음을 확인 할 수 있다. 물론 내용을 보면 각 상태를 저장하는 변수 name, surname으로 나뉘어져 있어 [1,2],[3,4]로 구분해서 생각할 수 있다. 그러나 지금은 1,2,3,4 순서에 집중해 보도록하자.

```jsx
// 순서 1
useState('Mary')
// 순서 2 
useEffect(persistForm)
// 순서 3
useState('Poppins')
// 순서 4
useEffect(updateTitle)
```

해당함수들은 '하나의 컴포넌트' 안에 있기 때문에, '랜더링이 일어날 때마다 컴포넌트 전체가 다시 작동'하게 된다. 여기서 주의해야 할 점은, 정해진 순서가 누락될 수 있다는 위험요소가 있다는 것이다. 그렇다면 순서가 누락될 위험요소는 무엇일까?

```jsx
    function Form() {
        // 1. 상태저장 Hook 호출(name state)
        const [name, setName] = useState('Mary');
        // -> 갱신(조건문 추가)
        // 2. 후처리 Hook 호출
        if (name !== '') {
            useEffect(function persistForm() {
                localStorage.setItem('formData', name);
            });
        }
        // 3. 상태저장 Hook 호출(surname state)
        const [surname, setSurname] = useState('Poppins');

        // 4. 후처리 Hook 호출
        useEffect(function updateTitle() {
            document.title = name + '' + surname;
        });
    }
```

갱신된 코드 내용에 주목하자. 조건문이 useEffect훅을 감싸고 있다. 해당 내용이 실행된다면 어떤일이 발생할까? 조건에 name 내용이 존재하지 않는다면, 어떻게 될까?

```jsx
// 순서 1
useState('Mary')

// 순서 2?? 
//useEffect(persistForm) 작동 건너뜀!!

// 순서 2 (이전 순서 3)
useState('Poppins') // 순서3번 이였던 hook이 2번으로 작동해서 오작동!!
// 순서 3 (이전 순서 4)
useEffect(updateTitle) // 순서4번 이였던 hook이 4번으로 작동해서 오작동!!
```

재밌게도 이전순서가 작동을 건너뛰자, 후순위 작동이였던 훅들이 앞 당겨졌다. 즉, '하나의 컴포넌트에서 랜더링이 일어날때마다 전체가 다시 작동한다는 원칙'이 전제로 깔려있었기 때문에, 인덱싱 된 순서가 지켜져야 되는데 그렇지 못해서 오작동이 발생 한 것이다.

보통은 상태가 저장된 변수의 갱신을 통해 해당 내용만 작동한다고 생각하지만, 사실은 컴포넌트 안의 상태변수가 어떤것으로 변하던, 전체가 갱신되는 일이 발생해서 생기는 것이다. 'React는 지정해 놓은 전체 Hooks를 순서대로 실행시킨다는 것을 기본전제라고 생각하자.' 그러면 훅 호출 자체를 조건문으로 감싸는 방식이 모순이 되니 사용할 일이 없어질 것이다.

그렇다면, 이 경우는 어떠할까?

```jsx
    // ...
    // 2. 후처리 Hook 호출
        useEffect(function persistForm() {
            if (name !== '') {
                localStorage.setItem('formData', name);
            }
        });
    // ...
```

Hook은 항상 실행시키되, name의 내용이 있다면, 후처리 하겠다는 것이다. 이것이 훅의 전체실행이라는 최소조건을 총족하되... 조건문, 반복문, 중첩함수를 사용할 수 있게 해주는 핵심요소 인 것이다.
